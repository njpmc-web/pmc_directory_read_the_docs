# Logs, Settings, and Permissions

## Goal

Review sign-in history, manage photo visibility/storage, and assign admin roles.

## Before you begin

- These functions require Super Admin.

## Steps

1. In **Login History**, review time, member, SMS/Email channel, and result/reason.
2. In **Settings**, review member-photo visibility and storage usage.
3. In **Permissions**, add/remove users for Super Admin, Pastor, and Lookup Admin.
4. Confirm each changed account sees only the intended menus.

## Successful result

Settings persist and roles see only menus defined in [Access by role](../access.md).

## Common problems

- **Removing your last Super Admin:** confirm another Super Admin first.
- **Storage unavailable:** retry later; contact support if it continues.
